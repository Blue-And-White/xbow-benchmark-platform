{# ===================== API 接入 ===================== #}

用 api-key 做题（核心流程）

在「我的解题表」建一张表后会得到一个 api-key。拿这一个 key 就能完成整个做题闭环——看题、启动、提交、关闭，无需登录态。下方 $KEY 替换为你的 api-key，$BASE 替换为平台地址。

1
看题目列表 + 本看板解题情况
curl -H "X-API-Key: $KEY" http://$BASE/api/challenges
# 返回 104 题，每题含 status: not_started | in_progress | abandoned | solved
2
启动一题（注入本次随机 flag，返回靶机地址）
curl -X POST -H "X-API-Key: $KEY" \
  http://$BASE/api/challenges/XBEN-001-24/start
# 返回: {"attempt_id", "url"(反代), "direct_url"(内部)}  靶机已就绪
📌 题目地址说明（重要）
  • url = 反代地址，格式 http://$BASE/c/<attempt_id>/ —— 公网可直接访问，这是你解题用的地址。平台内部把请求转发到靶机容器。
  • direct_url = http://127.0.0.1:<端口>/ —— 仅服务器内部可用，公网访问不了。仅供服务器本机调试用。
  • 所以：从公网/浏览器/agent 解题，一律用 url（反代）。direct_url 忽略即可。
3
访问靶机解题，拿到 flag（用返回的 url 反代地址）
curl http://$BASE/c/<attempt_id>/   # 公网反代, 直接访问
4
提交 flag — 正确则平台自动关闭并清理靶机容器（容器+数据卷+网络，无需手动停）
curl -X POST -H "X-API-Key: $KEY" -H "Content-Type: application/json" \
  -d '{"flag":"FLAG{...}"}' \
  http://$BASE/api/challenges/XBEN-001-24/submit
# 正确: {"correct":true,"solve_duration_ms":...}  -> 容器已清理
# 错误: {"correct":false}  -> 本次尝试保留，可继续解
5
放弃 / 手动关闭（解不出时，清掉本次 flag+计时，题回到未开始）
curl -X POST -H "X-API-Key: $KEY" \
  http://$BASE/api/challenges/XBEN-001-24/stop

{# ===================== MCP 接入 ===================== #}

MCP Server 接入

平台内置 MCP (Model Context Protocol) Server，AI agent（如 Claude、opencode 等）可通过标准 MCP 协议直接做题，无需手动调 HTTP API。

工作原理：MCP Server 运行在 agent 侧（本地），通过 stdin/stdout 与 agent 通信，通过 HTTP 调用平台 API。api-key 通过环境变量传入，agent 无需关心鉴权细节。

配置方法

在你的 MCP 客户端配置文件中加入：

{
  "mcpServers": {
    "xbow-ctf": {
      "command": "python",
      "args": ["path/to/mcp/server.py"],
      "env": {
        "XBOW_PLATFORM_URL": "http://$BASE",
        "XBOW_API_KEY": "$KEY"
      }
    }
  }
}

将 $BASE 替换为平台地址，$KEY 替换为你的 api-key。mcp/server.py 可从本仓库的 mcp/ 目录获取。

提供的 Tools

list_challenges

列出全部 104 题 + 当前状态（not_started / in_progress / abandoned / solved）。不暴露考点（tags/title/description）。

start_challenge

启动指定题目，返回靶机 URL（反代地址，公网可达）。每张解题表最多同时 3 题。

submit_flag

提交 flag。正确则自动关闭容器并记录用时；错误则保留本次尝试可继续解。

stop_challenge

手动放弃/关闭一道题。停止容器，作答记录保留。

典型做题流程（agent 视角）

1. list_challenges           → 看哪些题没做
2. start_challenge(XBEN-001-24) → 拿到靶机 URL
3. (访问靶机 URL 解题, 拿到 flag)
4. submit_flag(XBEN-001-24, "FLAG{...}") → 对了自动关
5. 重复 1-4 做下一题
