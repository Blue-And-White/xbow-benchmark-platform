用 api-key 做题（核心流程）

在「我的解题表」建一张表后会得到一个 api-key。拿这一个 key 就能完成整个做题闭环——看题、启动、提交、关闭，无需登录态。下方 $KEY 替换为你的 api-key。

1
看题目列表 + 本看板解题情况
curl -H "X-API-Key: $KEY" http://localhost:8000/api/challenges
# 返回 104 题，每题含 status: not_started | in_progress | solved
2
启动一题（注入本次随机 flag，返回靶机地址）
curl -X POST -H "X-API-Key: $KEY" \
  http://localhost:8000/api/challenges/XBEN-001-24/start
# {attempt_id, url(反代), direct_url}  -> 靶机环境已就绪
3
访问靶机解题，拿到 flag（用返回的 url 或 direct_url）
curl http://localhost:8000/c/<attempt_id>/   # 反代地址
4
提交 flag — 正确则平台自动关闭并清理靶机容器（容器+数据卷+网络，无需手动停）
curl -X POST -H "X-API-Key: $KEY" -H "Content-Type: application/json" \
  -d '{"flag":"FLAG{...}"}' \
  http://localhost:8000/api/challenges/XBEN-001-24/submit
# 正确: {"correct":true,"solve_duration_ms":...}  -> 容器已清理
# 错误: {"correct":false}  -> 本次尝试保留，可继续解
5
放弃 / 手动关闭（解不出时，清掉本次 flag+计时，题回到未开始）
curl -X POST -H "X-API-Key: $KEY" \
  http://localhost:8000/api/challenges/XBEN-001-24/stop
